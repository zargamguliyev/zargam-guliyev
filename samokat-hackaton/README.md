# Хакатон Самокат – Функция рекомендации рецептов

## Обзор проекта

Проект хакатона, посвящённый проектированию новой функции продукта для приложения доставки продуктов.

Цель — повысить вовлечённость пользователей в контент с рецептами.

---

## Проблема

Пользователи редко взаимодействуют с рецептами внутри приложения.

Выявленные проблемы:

- пользователи предпочитают сторонние сервисы рецептов
- текущий формат «историй» не интегрирован в основной пользовательский путь
- контент с рецептами используется недостаточно

---

## Бизнес-цель

Повысить вовлечённость пользователей в контент с рецептами и улучшить их удержание.

---

## Гипотеза

Если рекомендации рецептов будут интегрированы в корзину и формироваться на основе выбранных товаров, пользователи будут взаимодействовать с ними чаще.

---

## Решение TO-BE

Внедрение контекстуальных рекомендаций рецептов непосредственно в корзину.

Поведение системы:

- анализ товаров, добавленных в корзину
- сопоставление ингредиентов с базой данных рецептов
- предложение не менее 3 релевантных рецептов
- отображение недостающих ингредиентов
- возможность добавить недостающие ингредиенты в корзину

---

## Функциональные требования

Система должна:

1. анализировать содержимое корзины при изменении состава товаров
2. сопоставлять ингредиенты с базой данных рецептов
3. ранжировать рецепты по степени совпадения ингредиентов
4. отображать рекомендованные рецепты
5. поддерживать добавление недостающих ингредиентов в корзину

---

## Артефакты

- BPMN-модель процесса
- Диаграмма пользовательского потока (User Flow)
- Lo-Fi прототипы
- Презентация продуктовой концепции
